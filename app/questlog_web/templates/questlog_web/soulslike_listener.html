Death Detection

Template Matching DEFAULT
Lightweight pixel-pattern detection. Watches for the "YOU DIED" screen using image templates - no AI model, no large download, minimal CPU usage. Works out of the box.
EasyOCR (PyTorch) OPT-IN
AI-powered text recognition. More accurate across different resolutions and UI mods but requires a ~500MB PyTorch model download on first run and uses significantly more CPU. Enable in the setup window if template matching misses deaths.

Either way, F9 always logs a death manually if auto-detection misses one.

FAQ

Is this safe? Will it get me banned?
The Listener only reads your screen (OCR) and listens for key presses. It does not inject into the game or modify any game files. It is functionally equivalent to you manually pressing a button.
Where is my session token?
Start a Mortality Run from the QuestLog Souls Tracker page. Your session token is in the Web Tracker URL: questlog.casual-heroes.com/soulslike/runs/YOUR_TOKEN/
What's the difference between Template Matching and EasyOCR?
Template matching (default) is fast and lightweight - it looks for a pixel pattern of the "YOU DIED" screen. EasyOCR is AI-powered text recognition using PyTorch - more accurate but downloads ~500MB on first run and uses more CPU. Enable EasyOCR in the setup window only if template matching is missing deaths.
Can I use this on multiple monitors?
The OCR scans your primary monitor by default. Game window detection ensures it focuses on the right screen when the game is in the foreground.
